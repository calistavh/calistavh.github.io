GitHub Projects

Sprint 3 Week 3 | Dibuat: 5 Mei 2024

GitHub Projects adalah salah satu fitur GitHub untuk memvisualisasikan progress suatu proyek dalam sebuah organisasi atau tim. Tim dapat mendapatkan berbagai informasi dari visualisasi tersebut seperti perkembangan tugas-tugas yang perlu diselesaikan, beban pekerjaan yang dipegang setiap anggota, timeline proyek, dan sebagainya. Dengan demikian, fitur ini dapat memudahkan perencanaan dan meningkatkan produktivitas tim secara keseluruhan.

Kelompok TalaCare A1 menggunakan GitHub Project dalam mengembangkan game TalaCare, yang dapat diakses pada link ini. Berikut adalah penjelasan singkat untuk setiap view yang tersedia dalam GitHub Projects kami. Perlu ditambahkan bahwa GitHub Projects memiliki banyak kustomisasi sehingga tampilan atau fitur yang disebutkan selanjutnya bisa jadi berbeda antartim.

Board

Board adalah tempat untuk mengukur perkembangan issue (to-do item untuk mengimplementasikan atau memperbaiki suatu fitur) dalam proyek. Halaman ini adalah bagian yang paling sering kami gunakan, terutama saat sprint planning untuk merumuskan issue baru setiap awal sprint. Pada dasarnya terdapat tiga area untuk merepresentasikan status perkembangan issue, yaitu Todo, In Progress, dan Done.



Setiap issue dapat diisi estimasi beban atau story point, nomor iterasi atau sprint , jenis, prioritas, deskripsi, dan assignee(s) atau anggota tim yang bertanggung jawab menyelesaikan issue tersebut. Setiap issue yang bukan draft juga diberikan nomor untuk memudahkan perujukan issue tersebut, terutama saat mengajukan commit dan pull request. Saat keduanya mereferensi issue, informasi tersebut akan muncul dalam timeline issue seperti yang terlihat pada contoh di bawah.




Table

Sesuai namanya, table menunjukkan tabel berisi daftar issue dibagi berdasarkan iterasi atau sprint di sebelah kanan layar, serta jumlah issue yang dipegang setiap anggota tim di sebelah kiri layar. Halaman ini dapat berguna untuk memastikan bahwa pembagian tugas dilakukan secara rata.



Roadmap

Roadmap berisikan timeline aktifnya setiap issue dari kiri ke kanan. Rentang waktu issue dalam roadmap ditentukan oleh iterasi, dalam kasus kelompok kami dari tanggal awal hingga akhir sprint.
